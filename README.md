## English

### Overview

**Domina Pro** is a professional WordPress plugin for creating maintenance, update, coming soon, domain showcase and domain sale landing pages. It provides responsive templates, full Persian and RTL support, secure contact forms, administrator access controls, SEO-aware maintenance responses and extensive visual customization.

### Key Features

- Maintenance, update and coming soon pages
- Domain showcase and domain sale landing pages
- Ready-made, responsive and customizable templates
- Premium **Editorial Luxury** template
- Full Persian and RTL support
- Custom logo, favicon, hero image, colors and page content
- Secure contact form with validation, WordPress nonce, honeypot and rate limiting
- Authorized administrator bypass
- Exclusions for admin requests, AJAX, REST API, Cron and the WordPress login page
- Standard `503 Service Unavailable` response support
- Configurable `Retry-After` and `noindex` behavior
- Security headers and cache-control handling for temporary pages
- Contact details and social network links
- Conditional and optimized CSS/JavaScript loading
- Responsive layout for desktop, tablet and mobile devices

### Installation

1. Obtain the official plugin ZIP package.
2. In WordPress, go to **Plugins → Add New Plugin → Upload Plugin**.
3. Upload the ZIP package and activate Domina Pro.
4. Enter a valid license key.
5. Configure the template, content, branding, access rules and HTTP behavior.
6. Test the configuration on a staging environment before enabling it on a production website.

### Security

Domina Pro applies input validation, WordPress nonces, access controls, anti-spam protections, form rate limiting, approved template paths, proper HTTP maintenance responses and security headers to reduce common risks associated with temporary landing pages.

Please do not disclose security vulnerabilities in public issues. Report sensitive security findings through the official product support channel on Zhaket.

### Before Using Domina Pro

- Back up the website files and database before installation or updates.
- Test the plugin on a staging environment first.
- Verify administrator access, login routes and critical background processes after enabling maintenance mode.
- Purge WordPress, server and CDN caches after changing the maintenance status.
- Keep the plugin header version, release package and GitHub release version synchronized.

### Bug Reports and Contributions

When reporting an issue, include the following information without exposing sensitive data:

- WordPress version
- PHP version
- Domina Pro version
- Reproduction steps
- Relevant error message or log
- A short screenshot or screen recording

Improvement proposals and pull requests may be reviewed. The activation and licensing components must remain unchanged.
